# Структура директорий RUSH CLI

Проект RUSH CLI использует стандартизированную структуру путей, которая обеспечивает гибкость и удобство управления конфигурационными файлами и скриптами. Все пути определяются относительно переменной `$PREFIX`, которая по умолчанию устанавливается в `$HOME`.

## Основные директории

| Переменная     | Путь (по умолчанию)       | Описание                                                                  |
| :------------- | :------------------------ | :------------------------------------------------------------------------ |
| `$BINPATH`     | `$HOME/usr/bin`           | Зарезервировано для основных конфигурационных файлов (`conf`, `SEARCH-FETCH-CONFIG`), специфичных для каждой оболочки. |
| `$SCRIPTS`     | `$HOME/usr/scripts`       | Директория для пользовательских скриптов, создаваемых с помощью функции `creature`. |
| `$FILPATH`     | `$HOME/FIL/compiled`      | Зарезервированная директория для скомпилированных файлов или других специфических нужд проекта. |
| `$NOTES_FOLDER`| `$HOME/.notes`            | Директория для хранения текстовых заметок, создаваемых функцией `note`. |
| `$ALIAS_FILE`  | `$HOME/usr/aliases.sh`    | Файл, в котором хранятся пользовательские алиасы, созданные функцией `bandage`. |
| `$BACKUP_FOLDER`| `/sdcard/termux/`         | Директория для резервного копирования домашней директории.               |
| `$FLAG_FILE`   | `$HOME/FLAG`              | Флаг-файл, используемый для отслеживания первоначальной установки пакетов. |

## Пример структуры проекта

```
rush-cli/
├── .bashrc
├── .shrc
├── .zshrc
├── SETUP.sh
├── README.md
├── mkdocs.yml
├── docs/
│   ├── index.md
│   ├── installation.md
│   ├── features.md
│   └── structure.md
└── usr/
    ├── aliases.sh
    ├── bin/
    │   ├── conf
    │   ├── SEARCH-FETCH-CONFIG
    │   └── zsh/
    │       ├── conf
    │       └── SEARCH-FETCH-CONFIG
    └── scripts/
        └── # Ваши скрипты, созданные через creature
```

Эта структура обеспечивает четкое разделение между системными конфигурационными файлами, пользовательскими скриптами и данными, упрощая управление и обновление проекта.
