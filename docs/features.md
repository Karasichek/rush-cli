# Основные функции RUSH CLI

RUSH CLI предоставляет ряд мощных функций для улучшения вашего опыта работы в командной строке:

## Умный запуск файлов (`selection`)

Функция `selection` позволяет запускать скрипты и исполняемые файлы просто по их имени, без необходимости указывать полный путь или интерпретатор. Она автоматически определяет тип файла и использует соответствующий интерпретатор.

**Использование:**

```bash
selection <имя_файла> [аргументы...]
```

**Пример:**

Если у вас есть Python-скрипт `my_script.py` в вашей домашней директории, вы можете запустить его так:

```bash
selection my_script.py arg1 arg2
```

## Управление скриптами (`creature`)

Функция `creature` предназначена для быстрого создания новых пользовательских скриптов. Она создает файл скрипта в директории `$SCRIPTS` (по умолчанию `$HOME/usr/scripts`), делает его исполняемым и автоматически добавляет алиас для быстрого запуска.

**Использование:**

```bash
creature <имя_скрипта>
```

**Пример:**

```bash
creature my_new_script
```

Это создаст файл `$HOME/usr/scripts/my_new_script` и добавит алиас `my_new_script` в ваш конфигурационный файл оболочки.

## Интеграция с Git (`repo-check`, `repo-sync`)

RUSH CLI включает функции для упрощения работы с Git-репозиториями.

*   **`repo-check`**: Проверяет статус текущего Git-репозитория, сообщая, актуальна ли локальная ветка по отношению к удаленной, или есть ли новые коммиты.

    **Использование:**

    ```bash
    repo-check
    ```

*   **`repo-sync`**: Автоматизирует процесс синхронизации репозитория: добавляет все изменения, создает коммит, отправляет изменения в удаленный репозиторий и опционально создает Pull Request через `gh` (GitHub CLI).

    **Использование:**

    ```bash
bash
repo-sync <директория_репозитория> <сообщение_коммита> <заголовок_PR>
```

    **Пример:**

    ```bash
    repo-sync . "Обновление документации" "Добавить новую документацию"
    ```

## Система заметок (`note`)

Функция `note` позволяет быстро создавать и редактировать текстовые заметки. Заметки хранятся в директории `$NOTES_FOLDER` (по умолчанию `$HOME/.notes`). Если имя файла не указано, заметка будет названа по текущей дате.

**Использование:**

```bash
note [имя_файла]
```

**Примеры:**

```bash
note  # Создаст заметку типа 2026-04-28.txt
note MyProjectIdeas # Создаст заметку MyProjectIdeas.txt
```

## Создание алиасов (`bandage`)

Интерактивная функция `bandage` упрощает процесс создания и добавления алиасов в ваш конфигурационный файл. Она запрашивает имя алиаса, команду и описание, после чего автоматически добавляет его в `$ALIAS_FILE` (по умолчанию `$HOME/usr/aliases.sh`).

**Использование:**

```bash
bandage
```

**Пример интерактивного использования:**

```
╭
─ Creating alias ─
│ Alias name: ll
│ For command: ls -la
│ Some description: List all files in long format
╰
─
Алиас 'll' добавлен в /home/ubuntu/usr/aliases.sh
```

## Другие полезные алиасы и команды

*   **`blackout`**: Запускает новую сессию оболочки без загрузки конфигурационных файлов, предоставляя чистое окружение.
*   **`zshrc` / `bashrc` / `shrc`**: Быстро открывает основной конфигурационный файл текущей оболочки в текстовом редакторе (`helix` для Zsh, `micro` для Bash/sh).
*   **`config`**: Открывает файл общих настроек (`$BINPATH/conf` или `$BINPATH/zsh/conf`) в текстовом редакторе.
*   **`reload`**: Перезагружает конфигурацию текущей оболочки, применяя все последние изменения.
*   **`backup`**: Создает резервную копию домашней директории в `$BACKUP_FOLDER` (по умолчанию `/sdcard/termux/`).
*   **`backfrom`**: Восстанавливает данные из `$BACKUP_FOLDER` в домашнюю директорию.
*   **`site-check`**: Проверяет доступность и время отклика популярных веб-ресурсов (Google, GitHub, Yandex).
