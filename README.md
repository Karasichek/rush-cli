# RUSH CLI

**RUSH CLI** — это комплексный набор скриптов и конфигурационных файлов, разработанный General Spark Project для улучшения и автоматизации работы в командной строке. Проект предоставляет единую, симметричную и мощную среду для пользователей Zsh, Bash и других POSIX-совместимых оболочек (sh).

## Особенности

*   **Универсальность**: Поддержка `zsh`, `bash` и `sh` с единой логикой и симметричной структурой конфигурационных файлов.
*   **Автоматическая настройка**: Скрипт `SETUP.sh` и встроенные механизмы в `.zshrc`/`.bashrc`/`.shrc` автоматически устанавливают необходимые пакеты при первом запуске.
*   **Богатый набор утилит**: Включает в себя `tmux`, `helix`/`micro`, `rsync`, `eza`, `fastfetch`, `git`, `lazygit`, `lf`, `xmake` и другие инструменты для повышения продуктивности.
*   **Умный запуск файлов (`selection`)**: Функция `selection` позволяет запускать скрипты (Shell, Python, Perl, Ruby, Lua, Node.js) просто по имени файла, автоматически определяя интерпретатор по shebang или MIME-типу.
*   **Управление скриптами (`creature`)**: Быстрое создание новых пользовательских скриптов с автоматическим добавлением алиасов в конфигурацию оболочки.
*   **Интеграция с Git (`repo-check`, `repo-sync`)**: Удобные функции для проверки статуса репозиториев и быстрой синхронизации изменений (commit, push, создание PR через `gh`).
*   **Система заметок (`note`)**: Быстрое создание и редактирование текстовых заметок с автоматической генерацией имен файлов по дате.
*   **Кастомизация**: Настроенный `fastfetch` для красивого вывода системной информации и кастомные приглашения командной строки (PS1).

## Структура директорий

Проект использует стандартизированную структуру путей, определяемую переменной `$PREFIX` (по умолчанию `$HOME`):

*   `$BINPATH` (`$PREFIX/usr/bin`): Зарезервировано для конфигурационных файлов (`conf`, `SEARCH-FETCH-CONFIG`).
*   `$SCRIPTS` (`$PREFIX/usr/scripts`): Директория для пользовательских скриптов, создаваемых через `creature`.
*   `$FILPATH` (`$PREFIX/FIL`): Зарезервированная директория.
*   `$NOTES_FOLDER` (`$HOME/.notes`): Директория для хранения текстовых заметок.

## Установка

Для установки RUSH CLI выполните следующие шаги:

1.  Склонируйте репозиторий:
    ```bash
    git clone https://github.com/Karasichek/rush-cli.git
    ```
2.  Перейдите в директорию проекта и запустите скрипт установки:
    ```bash
    cd rush-cli
    chmod +x SETUP.sh
    ./SETUP.sh
    ```
    Скрипт `SETUP.sh` обновит список пакетов, установит необходимые зависимости (включая `lazygit` из официального репозитория, если он недоступен в пакетном менеджере) и скопирует конфигурационные файлы в вашу домашнюю директорию.

3.  Перезапустите вашу оболочку или выполните команду `reload` (если алиас уже доступен). При первом запуске оболочки (если файл `$HOME/FLAG` отсутствует) будет произведена дополнительная проверка и установка пакетов.

## Основные команды и алиасы

*   `blackout`: Запуск оболочки без загрузки конфигурационных файлов (чистое окружение).
*   `zshrc` / `bashrc` / `shrc`: Быстрое открытие основного конфигурационного файла текущей оболочки в текстовом редакторе.
*   `config`: Открытие файла общих настроек (`$BINPATH/conf` или `$BINPATH/zsh/conf`).
*   `reload`: Перезагрузка конфигурации текущей оболочки.
*   `backup`: Резервное копирование домашней директории в `/sdcard/termux/` (с использованием `rsync`).
*   `backfrom`: Восстановление данных из `/sdcard/termux/` в домашнюю директорию.
*   `creature <имя_скрипта>`: Создание нового скрипта в `$SCRIPTS` и добавление алиаса для него.
*   `selection <имя_файла> [аргументы...]`: Поиск и умный запуск файла.
*   `note [имя_файла]`: Создание или открытие заметки.
*   `repo-check`: Проверка статуса текущего Git-репозитория.
*   `repo-sync <директория> <сообщение_коммита> <заголовок_PR>`: Синхронизация репозитория и создание Pull Request.
*   `site-check`: Проверка доступности популярных веб-ресурсов (Google, GitHub, Yandex).

## Зависимости

RUSH CLI полагается на следующие пакеты, которые устанавливаются автоматически:

*   `tmux`
*   `helix` (для Zsh) / `micro` (для Bash/sh)
*   `xmake`
*   `rsync`
*   `eza`
*   `fastfetch`
*   `ncurses-utils`
*   `git`
*   `figlet`
*   `lf`
*   `lua5.4`
*   `lazygit`
*   `gh` (GitHub CLI)
